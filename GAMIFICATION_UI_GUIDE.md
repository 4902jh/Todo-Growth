# 게임화 UI 요소 가이드

## 개요
Todo Growth 프로젝트의 게임화 UI 요소는 사용자가 Todo를 달성하는 것을 게임처럼 재미있게 만들어 동기부여를 높이는 것을 목표로 합니다.

## 주요 게임화 UI 컴포넌트

### 1. CharacterDisplay 컴포넌트
캐릭터의 현재 상태를 시각적으로 표시하는 핵심 컴포넌트입니다.

#### 주요 요소

**1.1 레벨 배지 (Level Badge)**
```jsx
<div className="level-badge">Lv.{character.level}</div>
```
- **목적**: 사용자의 성장 단계를 한눈에 보여줌
- **디자인**: 둥근 배지 형태, 눈에 띄는 색상
- **심리적 효과**: 성취감과 진전감 제공

**1.2 캐릭터 아바타**
```jsx
<div className={`avatar-level-${Math.min(character.level, 10)}`}>
    🎮
</div>
```
- **목적**: 캐릭터의 시각적 표현
- **구현 아이디어**: 
  - 레벨에 따라 다른 이모지/이미지 사용
  - 레벨업 시 애니메이션 효과
  - 레벨이 높을수록 더 멋진 외형

**1.3 경험치 바 (Experience Bar)**
```jsx
<div className="exp-bar">
    <div 
        className="exp-bar-fill"
        style={{ width: `${character.experienceProgress}%` }}
    />
</div>
```
- **목적**: 다음 레벨까지의 진행도를 시각화
- **디자인 특징**:
  - 그라데이션 배경 (핑크→빨강)
  - 부드러운 애니메이션 전환
  - 퍼센트 표시로 정확한 진행도 제공
- **심리적 효과**: 
  - "조금만 더 하면 레벨업!" 동기부여
  - 진행감을 시각적으로 체감

**1.4 레벨업 안내**
```jsx
<div className="level-up-info">
    <p>경험치가 {character.requiredExperience} 이상이면 레벨업합니다!</p>
</div>
```
- **목적**: 레벨업 조건을 명확히 안내
- **디자인 특징**:
  - 간단한 텍스트 표시
  - 반투명 배경으로 강조
- **심리적 효과**:
  - 목표가 명확하여 동기부여
  - "조금만 더 하면 레벨업!" 느낌 제공

### 2. TodoItem 컴포넌트
각 Todo에 대한 게임화 요소를 포함한 컴포넌트입니다.

#### 주요 요소

**2.1 완료/실패 버튼**
```jsx
<button className="btn-complete" onClick={handleComplete}>
    ✅ 완료
</button>
<button className="btn-fail" onClick={handleFail}>
    ❌ 실패
</button>
```
- **목적**: 사용자가 Todo 상태를 직접 표시
- **디자인 특징**:
  - 완료 버튼: 보라색 그라데이션, 호버 시 확대 효과
  - 실패 버튼: 회색 배경, 경고 느낌
  - 클릭 시 즉시 피드백

**2.2 실시간 피드백 메시지**
```jsx
{message && (
    <div className={`message ${message.type}`}>
        {message.text}
    </div>
)}
```
- **메시지 타입**:
  - **Success**: "✅ 완료! 경험치 +5 획득!"
  - **Level Up**: "🎉 레벨업! 경험치 +5 획득!"
  - **Warning**: "❌ 실패! 경험치 -5"
  - **Level Up (실패 후)**: "⚠️ 레벨업! 경험치 -5 감소"
- **디자인 특징**:
  - 슬라이드 인 애니메이션
  - 타입별 색상 구분 (초록/노랑/빨강)
  - 3초 후 자동 사라짐
- **심리적 효과**:
  - 즉각적인 보상/패널티 피드백
  - 행동의 결과를 명확히 인지

**2.3 완료 상태 표시**
```jsx
<div className="completed-badge">
    오늘 완료됨! 🎉
</div>
```
- **목적**: 오늘 완료한 Todo을 명확히 표시
- **디자인 특징**:
  - 배경색 변경 (보라색 그라데이션)
  - 축하 이모지와 함께 표시
  - 버튼 대신 배지로 교체

## 게임화 UI 디자인 원칙

### 1. 즉각적인 피드백
- 사용자의 행동에 대해 즉시 시각적 피드백 제공
- 경험치 획득, 레벨업 등의 이벤트를 명확히 표시

### 2. 진행도 시각화
- 진행 바를 통해 목표까지의 거리를 명확히 표시
- 숫자와 퍼센트를 함께 사용하여 정확성 제공

### 3. 긍정적 강화
- 성공 시 밝고 화려한 색상 사용
- 축하 메시지와 이모지로 성취감 강조

### 4. 명확한 피드백
- 실패 시 경험치 감소를 명확히 표시
- 레벨업 가능 여부를 시각적으로 표현

### 5. 지속적인 동기부여
- 레벨업 시 보상으로 성장감 제공
- 경험치 바로 목표까지의 거리 시각화

## CSS 애니메이션 효과

### 경험치 바 애니메이션
```css
.exp-bar-fill {
    transition: width 0.3s ease;
}
```
- 경험치 획득 시 부드럽게 증가하는 효과

### 레벨업 축하 효과 (향후 구현 가능)
```css
.level-up-animation {
    animation: celebrate 0.5s ease;
}

@keyframes celebrate {
    0%, 100% { transform: scale(1); }
    50% { transform: scale(1.2); }
}
```
- 레벨업 시 캐릭터가 확대되는 효과

### 메시지 슬라이드 인
```css
@keyframes slideIn {
    from {
        opacity: 0;
        transform: translateY(-10px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}
```
- 피드백 메시지가 부드럽게 나타나는 효과

## 추가 개선 아이디어

### 1. 레벨업 축하 애니메이션
- 레벨업 시 화면 전체 축하 애니메이션
- 컨페티 효과나 파티클 애니메이션

### 2. 캐릭터 성장 시각화
- 레벨에 따른 캐릭터 외형 변화
- 애니메이션 GIF 또는 SVG 사용

### 3. 달성률 차트
- 주간/월간 달성률 그래프
- 진행 추세를 시각적으로 표시

### 4. 업적 시스템 UI
- 달성한 업적을 배지 형태로 표시
- "7일 연속 달성", "레벨 10 달성" 등

### 5. 사운드 효과 (선택사항)
- 경험치 획득 시 효과음
- 레벨업 시 축하 사운드

## 사용자 경험 플로우

### Todo 완료 시나리오
1. 사용자가 "✅ 완료" 버튼 클릭
2. 로딩 상태 표시 ("처리 중...")
3. 성공 메시지 표시 ("✅ 완료! 경험치 +5 획득!")
4. CharacterDisplay 컴포넌트 자동 업데이트
   - 경험치 바 증가
   - 레벨업 시 레벨 배지 업데이트 및 경험치 초기화
5. 3초 후 메시지 자동 사라짐

### 레벨업 시나리오
1. Todo 완료로 경험치 +5 획득
2. 경험치가 20 이상이 되면 레벨업 조건 달성
3. 특별 메시지 표시 ("🎉 레벨업!")
4. 캐릭터 상태 업데이트
   - 레벨 +1
   - 경험치 = 0으로 초기화
5. 축하 애니메이션 (선택사항)

### Todo 실패 시나리오
1. 사용자가 "❌ 실패" 버튼 클릭
2. 확인 다이얼로그 표시
3. 실패 처리 후 경고 메시지 ("❌ 실패! 경험치 -5")
4. CharacterDisplay 업데이트
   - 경험치 바 감소
   - 경험치가 20 이상이면 레벨업 가능 (레벨업 메시지 표시)

## 접근성 고려사항

1. **색상 대비**: 색상만으로 정보를 전달하지 않고 텍스트/아이콘 함께 사용
2. **키보드 네비게이션**: 모든 버튼이 키보드로 접근 가능
3. **스크린 리더**: 적절한 ARIA 레이블 사용
4. **애니메이션 제어**: 사용자 설정에 따라 애니메이션 비활성화 옵션

## 성능 최적화

1. **이미지 최적화**: 캐릭터 이미지는 WebP 형식 사용
2. **애니메이션 최적화**: CSS transform 사용 (GPU 가속)
3. **상태 업데이트**: 불필요한 리렌더링 방지 (React.memo 활용)
4. **API 호출 최적화**: 디바운싱/캐싱 활용
